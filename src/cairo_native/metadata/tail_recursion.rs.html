1//! # Tail recursion information
2//!
3//! Whenever the compiler detects a direct tail-recursive function call it'll insert this metadata.
4//! If the libfunc handler decides to use it by setting a return target, the compiler will insert
5//! the required instructions to make it really tail-recursive.
6//!
7//! Directly recursive functions are detected by checking if the current statement is a function
8//! call. Indirect recursion is not handled and generates normal recursive code.
9//!
10//! Next, the compiler check whether those direct recursive calls are in fact tail recursive.
11//! Recursive function calls are tail recursive if nothing declared before the function call is used
12//! after it. Due to the way Sierra works, this is as simple as checking whether the state is empty
13//! after taking the variables sent to the function call from itself.
14//!
15//! When tail recursion is detected, a counter is added which counts the current recursion depth.
16//! The counter being zero means that no recursion has taken place (either because the program
17//! hasn't reached the recursion point yet, or because it has completely unwinded already).
18//!
19//! Every time the recursive function call is invoked, the libfunc builder should increment this
20//! counter by one and jump to the recursion target block provided by the meta. This recursion
21//! target is provided by the compiler and should point to the beginning of the function and have
22//! the same arguments. The function call libfunc builder should then set the return target to a
23//! block which jumps to the next libfunc (a standard libfunc builder terminator as in every other
24//! libfunc).
25//!
26//! When the compiler generates the code for a return statement of a tail recursive function, it'll
27//! first check whether the depth counter is zero or not. If zero, a normal return statement will be
28//! generated since it'd mean our parent frame is a different function. However if the counter is
29//! not zero, the counter is decremented by one and a jump to the return target is generated, which
30//! will return control to the function call libfunc builder. The builder should then jump to the
31//! next libfunc statement as if it were a normal function call.
32//!
33//! After calling the libfunc builder, the metadata is removed from storage to avoid collisions
34//! later on.
35//!
36//! The same algorithm can be applied multiple times if there are multiple tail-recursive calls
37//! within a function. The compiler should create a different depth counter for each recursive call
38//! in the function.
39
40use melior::ir::{Block, BlockRef, Value, ValueLike};
41use mlir_sys::{MlirBlock, MlirValue};
42
43/// The tail recursion metadata.
44///
45/// Check out [the module](self) for more information about how tail recursion works.
46// TODO: Find a way to pass stuff with lifetimes while keeping the compiler happy.
47#[derive(Debug)]
48pub struct TailRecursionMeta {
49    depth_counter: MlirValue,
50
51    recursion_target: MlirBlock,
52    return_target: Option<MlirBlock>,
53}
54
55impl TailRecursionMeta {
56    /// Create the tail recursion meta.
57    pub fn new(depth_counter: Value, recursion_target: &Block) -> Self {
58        Self {
59            depth_counter: depth_counter.to_raw(),
60            recursion_target: recursion_target.to_raw(),
61            return_target: None,
62        }
63    }
64
65    /// Return the current depth counter value.
66    pub fn depth_counter<'ctx, 'this>(&self) -> Value<'ctx, 'this> {
67        unsafe { Value::from_raw(self.depth_counter) }
68    }
69
70    /// Return the recursion target block.
71    pub fn recursion_target<'ctx, 'this>(&self) -> BlockRef<'ctx, 'this> {
72        unsafe { BlockRef::from_raw(self.recursion_target) }
73    }
74
75    /// Return the return target block, if set.
76    pub fn return_target<'ctx, 'this>(&self) -> Option<BlockRef<'ctx, 'this>> {
77        self.return_target
78            .map(|ptr| unsafe { BlockRef::from_raw(ptr) })
79    }
80
81    /// Set the return target block.
82    pub fn set_return_target(&mut self, block: &Block) {
83        self.return_target = Some(block.to_raw());
84    }
85}
